Testing Long-Running Webhooks

This page helps you test WebSophon's ability to handle slow webhook responses.

‚ö†Ô∏è Note: WebSophon now supports webhook requests up to 5 minutes (300 seconds) without timing out.

Webhook Testing Services

Use these services to simulate slow responses:

1. Webhook.site with Delay

Create a webhook at webhook.site and add delay:

https://webhook.site/YOUR-UUID?delay=30

Replace YOUR-UUID with your unique webhook URL. The delay=30 parameter adds a 30-second delay.

2. HTTPBin Delay Endpoint

Use HTTPBin's delay endpoint (delays up to 10 seconds):

https://httpbin.org/delay/10

3. Custom n8n Workflow with Wait Node

Create an n8n workflow with a Wait node to simulate processing time.

What to Expect

  1. Immediate Event Creation: Event appears with "‚è≥ Waiting for response..." status
  2. Screenshot Available: You can view the captured screenshot right away
  3. Request Data Visible: Check what was sent to the webhook
  4. Response Section Shows Pending: Blue box indicates waiting for response
  5. Auto-Update: When response arrives, the event updates automatically
  6. Field Results Update: Field evaluations appear and last results refresh

Test Scenarios

Scenario 1: Normal Delay (30 seconds)

  • Use a 30-second delay
  • Verify event shows as pending
  • Confirm auto-update when response arrives

Scenario 2: Very Long Delay (2 minutes)

  • Use a 120-second delay
  • Test multiple concurrent captures
  • Verify all events update independently

Scenario 3: Timeout Test (>5 minutes)

  • Try a delay longer than 300 seconds
  • Should see timeout error after 5 minutes
  • Event should update with error status

Debugging Tips

  • Open Chrome DevTools (F12) to see console logs
  • Watch for "Tracking event" and "Updating event" messages
  • Check Network tab to see webhook requests
  • Multiple extension popups can be open to see real-time updates
